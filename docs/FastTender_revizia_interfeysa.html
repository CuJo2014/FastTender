Ревизия интерфейса страницы спецификации

Страница: /specifications/:id Стиль: Clean SaaS (navy/blue, Geist) Статус: к реализации

Документ описывает изменения интерфейса страницы спецификации: разбор текущих проблем, приоритеты и два готовых интерактивных мокапа (сводка-метрик и таблица результатов), встроенных ниже. Мокапы рабочие — действия, фильтры и пересчёты можно проверить прямо в документе.

Блок 1 — Сводка-метрик

Две оси вместо одной строки чисел

Текущая сводка сводит пять разнородных чисел в один ряд, где уверенность модели, статус матчинга и статус workflow перемешаны. Ключевое следствие — ≥90% = 0 при шести подтверждённых строках выглядит как ошибка. Новая версия разделяет два независимых измерения:

  • Прогресс верификации — сколько строк закрыто (workflow). Прогресс-бар и счётчики «закрыто / осталось».
  • Качество сопоставления — распределение по уверенности как пропорциональная полоса. Прежний «Не найдено» разделён на < 50% (слабый кандидат есть) и Нет кандидата, поэтому суммы сходятся: 0 + 10 + 8 + 33 = 51.
  • Легенда = фильтр — клик по категории затемняет остальные сегменты и ведёт к фильтрации таблицы.

Блок 2 — Таблица результатов

Верификация без лишних модалок

Таблица перестроена под основную задачу — быстрый разбор очереди сопоставлений. Действия вынесены в строку, цветовые сигналы согласованы со статусом, добавлены фильтр, сортировка и массовые операции.

  • Инлайн ✓ / ✗ в колонке «Решение»; подтверждённая строка показывает статус и возврат в работу (↩); строка без кандидата предлагает «Подобрать».
  • Согласование цвета — на решённых строках score становится приглушённым, чтобы красный процент не спорил с зелёным «Подтверждено».
  • Фильтр и сортировка — сегменты по статусу (та же ось, что в легенде сводки) и сортировка по уверенности для триажа.
  • Массовый выбор — чекбоксы + bulk-бар; «Авто-подтвердить (N)» со счётчиком и дизейблом при нуле.

Как проверить в мокапе

Подтвердите жёлтую/красную строку — увидите приглушение бейджа. Снизьте «Порог» с 0.90 до 0.60 — счётчик у «Авто-подтвердить» пересчитается и кнопка станет активной. Переключите сегменты статуса и сортировку.

Заметки по интеграции

  • Данные. Счётчики сегментов, проценты прогресса и доли категорий в мокапах захардкожены/считаются по показанному срезу — на проде берутся из бэкенда. У сегментов распределения flex = количество в категории; пустая категория (flex:0.001) даёт почти невидимую полоску.
  • Логика бейджа уверенности. Приглушение завязано на status ∈ {confirmed, rejected} — ложится на серверный ПриЗаписи-флоу с аудитом. conf = null рендерится как —.
  • Связка сводка → таблица. Фильтр легенды и сегментный фильтр таблицы — одна ось состояния; разумно гонять через общий параметр (категория уверенности / статус).
  • Действия строки. ✓ подтверждает текущего кандидата из «Выбранной позиции»; ✗ → rejected; «Подобрать» / «Кандидаты» открывают модалку выбора. Подтверждение пишет запись аудита.
  • Авто-подтверждение. Цель = строки pending && conf ≥ порог·100; каждое подтверждение проходит тем же путём аудита, что и ручное.
  • Доступность. Видимый :focus-visible, aria-label на чекбоксах и иконочных кнопках, prefers-reduced-motion уважается. Шрифт Geist через Google Fonts с системным фоллбэком.
