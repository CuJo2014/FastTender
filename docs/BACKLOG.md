## Парсинг цен (разнобой у поставщиков)

**Проблема.** В прайсах поставщиков колонка с ценой называется по-разному
и часто не совпадает с текущими синонимами PRICE:
- TEL (Техэлектро): цена не определилась — заголовок неизвестен (нужно увидеть файл)
- Может встречаться: `РРЦ`, `Цена розничная`, `Розн. цена`, `Цена реализации`,
  `Отпускная цена`, `Прайс`, `Цена прайс`, `Цена опт.`, `Стоимость единицы`,
  `Цена с НДС` vs `Цена без НДС` (две колонки — какую брать?)
- Иногда цена в формате с символом валюты (`100 ₽`), форматирование `1 234,56`
- Может быть несколько ценовых колонок (опт/розн/акция) — нужен выбор

**Что делать.**
1. Собрать примеры заголовков из всех загруженных прайсов (`TEL`, `INT`, `AKR`...)
2. Расширить `COLUMN_SYNONYMS[SpecField.PRICE]`
3. Если есть несколько ценовых колонок — продумать UX выбора (через UI «Особенности прайса» — `preferred_price_column: "розничная"`)
4. Парсер для строк `"100 ₽"` уже частично работает (parse_decimal), но проверить кейсы с пробелами как разделителями тысяч

Текущие синонимы PRICE:
```
"цена", "стоимость", "цена за единицу", "цена за ед",
"price", "unit price", "cost"
```

   - API: эндпоинты клиентов (`/clients`...) + PATCH `/specifications/{id}`
